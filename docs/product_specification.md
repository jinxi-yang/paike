# 北清商学院智能排课系统 - 产品说明书

## 1. 产品概述

### 1.1 产品简介
北清商学院智能排课系统是一款专为商学院教育机构设计的智能化课程调度管理系统。该系统采用先进的AI算法和自动化排课技术，帮助教务管理人员高效完成复杂的课程安排工作，显著提升教学资源配置效率。

### 1.2 产品定位
- **目标用户**: 商学院教务管理人员、教学组织者
- **应用场景**: 学院日常教学管理、培训班课程安排、师资调配
- **核心价值**: 通过智能化手段解决传统人工排课的效率低下、易出错、资源分配不合理等问题

### 1.3 产品愿景
打造国内领先的智能化教育排课解决方案，成为商学院数字化教学管理的标准工具。

## 2. 核心功能

### 2.1 智能排课引擎
#### 2.1.1 自动排课算法
- **多维度约束处理**: 综合考虑节假日、讲师档期、班主任可用性等约束条件
- **冲突检测与解决**: 智能识别并处理时间冲突、师资冲突等问题
- **优化调度策略**: 基于启发式算法实现课程时间的最优化分配

#### 2.1.2 排课规则配置
- **时间间隔控制**: 支持4周间隔的课程安排规则
- **节假日规避**: 集成国家法定节假日数据库，自动避开节假日排课
- **师资负载均衡**: 合理分配讲师授课任务，避免过度集中

### 2.2 基础数据管理
#### 2.2.1 培训班类型管理
- 支持多种培训班类型的定义和管理
- 每种类型可配置专属的课程体系和排课规则
- 提供标准化的培训班模板

#### 2.2.2 课题管理体系
- 结构化课题分类管理
- 支持课题序列化编排
- 课题与讲师、课程的灵活关联

#### 2.2.3 师资管理
- **讲师信息维护**: 完整的讲师档案管理
- **专业能力标注**: 讲师专长领域标识
- **授课记录追踪**: 历史授课情况统计分析

#### 2.2.4 班主任管理
- 班主任信息维护
- 班级责任归属管理
- 班主任工作量统计

### 2.3 课程调度管理
#### 2.3.1 班级管理
- 班级创建与信息维护
- 自动课表生成功能
- 班级状态跟踪管理

#### 2.3.2 课表调整
- **单课调整**: 支持课程时间、讲师的灵活调整
- **批量调整**: 支持多个课程的同时调整
- **冲突预警**: 调整前自动检测潜在冲突

#### 2.3.3 合班功能
- 多班级相同课题的合并上课
- 合班后的资源整合优化
- 合班记录追溯管理

### 2.4 月度计划管理
#### 2.4.1 计划制定
- 基于约束条件的月度排课计划自动生成
- 支持人工干预和调整
- 计划版本管理和对比

#### 2.4.2 发布管理
- 计划审核流程
- 多渠道通知机制
- 发布状态跟踪

### 2.5 AI智能辅助
#### 2.5.1 智能决策支持
- AI决策引擎
- 复杂约束条件的智能解析
- 最优解推荐算法

#### 2.5.2 冲突分析报告
- 详细的冲突原因分析
- 智能化解建议
- 决策支持信息

## 3. 技术架构

### 3.1 系统架构
```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   前端界面层    │◄──►│   API服务层      │◄──►│   数据存储层    │
│  (HTML+JS)     │    │   (Flask)        │    │  (SQLite)       │
└─────────────────┘    └──────────────────┘    └─────────────────┘
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│  用户交互界面   │    │  业务逻辑处理    │    │  数据持久化     │
│                 │    │                  │    │                 │
└─────────────────┘    └──────────────────┘    └─────────────────┘
```

### 3.2 技术栈
- **前端技术**: HTML5, JavaScript, Tailwind CSS
- **后端框架**: Python Flask
- **数据库**: SQLite (支持MySQL扩展)
- **AI平台**: Dify智能体平台
- **节假日服务**: timor.tech API + 本地缓存

### 3.3 核心组件
#### 3.3.1 后端服务模块
- `app.py`: 主应用入口和服务器启动
- `models.py`: 数据库模型定义
- `config.py`: 系统配置管理
- `routes/`: RESTful API路由模块
  - `schedule.py`: 排课调度核心逻辑
  - `classes.py`: 班级管理接口
  - `teacher.py`: 讲师管理接口
  - `topic.py`: 课题管理接口
  - `ai.py`: AI智能体接口

#### 3.3.2 数据模型设计
- **TrainingType**: 培训班类型
- **Topic**: 课题信息
- **Teacher**: 讲师档案
- **Homeroom**: 班主任信息
- **Course**: 课程详情
- **TeacherCourseCombo**: 教-课组合
- **Class**: 班级信息
- **ClassSchedule**: 课表安排
- **MonthlyPlan**: 月度计划

## 4. 业务流程

### 4.1 标准排课流程
1. **数据准备**: 录入培训班类型、课题、讲师等基础数据
2. **班级创建**: 新建班级并设定开班日期
3. **自动排课**: 系统根据规则自动生成初步课表
4. **人工调整**: 教务人员对课表进行微调优化
5. **冲突处理**: 系统检测并协助解决各类冲突
6. **计划发布**: 最终课表审核通过后正式发布

### 4.2 异常处理机制
- **节假日冲突**: 自动跳过节假日，寻找合适时间
- **师资冲突**: 提供备选讲师建议或时间调整方案
- **班主任冲突**: 协助协调班主任工作安排
- **时间冲突**: 智能推荐替代时间段
