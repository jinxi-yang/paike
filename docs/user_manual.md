## 系统简介

北清商学院智能排课系统是一个集成了人工智能技术的教学管理平台，专门用于解决商学院复杂的课程安排问题。系统能够自动处理各种约束条件，生成最优化的课表安排。

### 主要优势
- 🤖 **AI智能排课**: 自动处理复杂的时间和资源约束
- ⚡ **高效便捷**: 大幅减少人工排课时间
- 🛡️ **冲突预防**: 提前识别并解决潜在冲突
- 📊 **数据可视**: 清晰的界面展示排课结果

## 快速开始

### 系统要求
- 现代浏览器（Chrome、Firefox、Edge等）
- 网络连接（用于节假日数据获取）

### 访问方式
1. 确保后端服务已启动（默认端口5000）
2. 在浏览器中打开 `frontend/index.html` 文件
3. 系统将自动加载并显示主界面

## 功能详解

### 主要功能模块

#### 1. 培训班管理
**功能描述**: 管理不同类型的培训班信息
- 查看所有培训班类型及其课程结构
- 添加新的培训班类型
- 修改培训班基本信息

**操作路径**: 主界面 → 培训班标签页

#### 2. 班级管理
**功能描述**: 创建和管理具体班级
- 创建新班级并指定开班日期
- 查看班级详细信息和课表
- 调整班级状态和属性

**操作路径**: 主界面 → 班级标签页

#### 3. 课表查看与调整
**功能描述**: 查看和修改具体课程安排
- 按月/周视图查看课表
- 调整课程时间安排
- 更换授课讲师
- 处理课程冲突

**操作路径**: 主界面 → 课表标签页

#### 4. 月度计划
**功能描述**: 制定和发布月度教学计划
- 生成月度排课计划
- 审核和调整计划内容
- 正式发布教学安排

**操作路径**: 主界面 → 月度计划标签页

#### 5. AI辅助决策
**功能描述**: 利用AI技术辅助复杂排课决策
- 输入约束条件获得智能建议
- 分析冲突原因并提供解决方案
- 优化排课策略

**操作路径**: 主界面 → AI助手标签页

## 操作指南

### 基础操作流程

#### 创建新班级
1. 进入"班级"标签页
2. 点击"添加班级"按钮
3. 填写班级基本信息：
   - 选择培训班类型
   - 输入班级名称
   - 选择班主任
   - 设置开班日期
4. 点击"创建"，系统将自动生成课表

#### 调整课程安排
1. 进入"课表"标签页
2. 找到需要调整的课程
3. 点击课程卡片上的"调整"按钮
4. 在弹窗中：
   - 选择新的日期（必须是周六）
   - 选择新的讲师组合
   - 添加备注说明
5. 点击"保存"完成调整

#### 处理课程冲突
1. 系统会自动检测冲突并在界面上标红显示
2. 点击冲突课程查看详情
3. 根据系统提供的建议进行调整：
   - 更换讲师
   - 调整时间
   - 合班处理

#### 生成月度计划
1. 进入"月度计划"标签页
2. 选择目标月份
3. 点击"生成计划"按钮
4. 系统会根据当前班级情况自动生成计划
5. 审核计划内容，必要时进行手动调整
6. 确认无误后点击"发布计划"

### 高级功能使用

#### 合班操作
当多个班级需要上同一门课时：
1. 在课表界面选择相关课程
2. 点击"合班"功能
3. 选择合并后的上课时间
4. 系统会自动处理资源合并

#### 批量调整
需要同时调整多个课程时：
1. 使用筛选功能找到目标课程
2. 批量选择需要调整的课程
3. 统一进行时间或讲师调整

## 常见问题

### Q1: 为什么某些日期无法排课？
**A**: 可能原因包括：
- 该日期是法定节假日
- 班主任在该日期不可用
- 相关讲师已被其他课程占用
- 系统检测到时间冲突

### Q2: 如何处理讲师请假的情况？
**A**: 
1. 在调整课程时选择其他可用讲师
2. 或将课程推迟到讲师可以上课的时间
3. 系统会在调整时自动检测讲师档期

### Q3: 合班功能有什么注意事项？
**A**: 
- 只能合并相同课题的课程
- 需要确保合并后的教室容量足够
- 合班后原班级的进度会同步调整

### Q4: 系统如何处理节假日？
**A**: 
- 系统内置了节假日数据库
- 排课时会自动避开节假日
- 支持自定义节假日设置

### Q5: AI建议总是合理的吗？
**A**: 
- AI建议基于当前数据和规则
- 复杂情况下建议人工复核
- 可以根据实际情况调整AI的建议

## 最佳实践

### 排课策略建议
1. **提前规划**: 尽早录入基础数据，给系统充足的时间优化
2. **分批处理**: 不要一次性创建过多班级，建议分批次进行
3. **预留缓冲**: 在重要时间节点前后预留调整空间
4. **定期检查**: 定期查看冲突报告，及时处理问题

### 数据管理建议
1. **信息完整**: 确保讲师、班主任等基础信息准确完整
2. **及时更新**: 人员变动或时间调整要及时在系统中更新
3. **备份重要**: 定期备份重要的排课数据
4. **权限控制**: 根据职责分配不同的操作权限

### 冲突处理原则
1. **优先级排序**: 重要课程优先安排
2. **就近调整**: 优先考虑时间相近的调整方案
3. **资源平衡**: 避免某个讲师或教室过度集中
4. **沟通协调**: 重大调整前与相关人员充分沟通

### 系统维护建议
1. **定期重启**: 建议每周重启一次服务清理缓存
2. **数据清理**: 定期清理过期的历史数据
3. **性能监控**: 关注系统响应速度，必要时优化
4. **版本更新**: 及时更新到最新稳定版本
