# 北清商学院排课系统 - 数据字典

> 本文档记录所有数据库表结构及字段含义，供开发和维护参考。

---

## 1. project - 项目（培训班类型）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键，自增 |
| name | VARCHAR(100) | ✅ | 项目名称（唯一），如"领袖增长本科班" |
| description | TEXT | | 项目描述 |
| created_at | DATETIME | | 创建时间 |

**关联关系**: 一个项目下有多个课题(topic)和多个班级(class)

---

## 2. topic - 课题（教学模块）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键，自增 |
| project_id | INTEGER | ✅ | 所属项目ID → project.id |
| sequence | INTEGER | ✅ | 课题顺序(1-8)，每个项目有8个课题 |
| name | VARCHAR(200) | ✅ | 课题名称 |
| is_fixed | BOOLEAN | | 是否固定课题（首课/结课不可调换） |
| description | TEXT | | 课题描述 |

**关联关系**: 属于一个项目(project)，下挂多个教-课组合(teacher_course_combo)

| 字段 | 类型 | 必填 | 说明 |

| project_id | INTEGER | ✅ | 所属项目ID → project.id |

## 8. class_schedule - 班级课表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键，自增 |
| class_id | INTEGER | ✅ | 班级ID → class.id |he
| topic_id | INTEGER | ✅ | 课题ID → topic.id |
| combo_id | INTEGER | | 教-课组合ID(Day1) → teacher_course_combo.id |
| combo_id_2 | INTEGER | | 教-课组合ID(Day2) → teacher_course_combo.id |
| scheduled_date | DATE | ✅ | 排课日期(通常为周六) |
| week_number | INTEGER | | 第几周（课次序号） |
| status | VARCHAR(20) | | 状态: scheduled(已排课)/completed(已完成)/cancelled(已取消)，默认scheduled。冲突通过conflict_type字段标记，合班通过merged_with字段标记 |
| conflict_type | VARCHAR(20) | | 冲突类型: teacher(讲师冲突)/homeroom(班主任冲突)/holiday(节假日)/manual(手动标记)。有值即表示存在冲突 |
| notes | TEXT | | 备注 |
| merged_with | INTEGER | | 合班标识(指向主课表ID)。有值表示该记录已被合并到主记录，拆分时清除 |
| merge_snapshot | TEXT | | 合班前快照(JSON)，保存合班前的原始日期/组合/状态，拆分时用于恢复 |
| homeroom_override_id | INTEGER | | 本次排课临时班主任ID → homeroom.id。为空时使用班级默认班主任 |
| created_at | DATETIME | | 创建时间 |
| updated_at | DATETIME | | 更新时间 |

**业务含义**: 核心排课表，每条记录 = 某班某课题在某天的排课安排

**状态说明**:
- `scheduled`：默认状态，表示已排课（含预览/草稿阶段）
- `completed`：课程日期已过，自动标记为已完成
- `cancelled`：已取消的排课记录

**冲突与合班**（不再使用独立状态，通过字段标记）:
- 冲突：`status` 保持 `scheduled`，通过 `conflict_type` 字段标记冲突类型
- 合班：`status` 保持 `scheduled`，通过 `merged_with` 字段指向主课表记录ID

**状态流转**:
- `scheduled` → `completed`（课程日期过后自动标记）
- `scheduled` → `cancelled`（手动取消）

**发布逻辑**: 发布/取消发布月度计划仅修改 `MonthlyPlan.status`，不再改变课表记录状态

## 9. monthly_plan - 月度计划

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键，自增 |
| year | INTEGER | ✅ | 年份 |
| month | INTEGER | ✅ | 月份 |
| status | VARCHAR(20) | | draft(草稿)/published(已发布) |
| published_at | DATETIME | | 发布时间 |
| created_at | DATETIME | | 创建时间 |
| updated_at | DATETIME | | 更新时间 |

**约束**: year + month 联合唯一

---

## 10. schedule_constraint - 排课约束条件

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键，自增 |
| monthly_plan_id | INTEGER | ✅ | 所属月度计划ID → monthly_plan.id |
| constraint_type | VARCHAR(30) | | 类型: teacher_unavailable(讲师不可用)/blocked_date(日期锁定)/custom(自定义) |
| description | TEXT | ✅ | 原始描述文本（用户输入） |
| parsed_data | TEXT | | AI解析后的JSON数据 |
| is_active | BOOLEAN | | 是否启用，默认true |
| created_at | DATETIME | | 创建时间 |

## ER关系图

```
project 1──N topic 1──N teacher_course_combo N──1 teacher
   │                        │
   │                    N──1 course
   │
   1──N class 1──N class_schedule
          │              │
      N──1 homeroom   N──1 topic
                         │
                     N──1 teacher_course_combo (Day1: combo_id)
                     N──1 teacher_course_combo (Day2: combo_id_2)
                     N──0..1 homeroom (临时班主任: homeroom_override_id)

monthly_plan 1──N schedule_constraint
```
