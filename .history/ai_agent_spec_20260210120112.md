# 外部智能体对接规范 (AI Agent Integration Spec)

本文档详细说明了“北清商学院智能排课系统”与外部AI智能体的交互协议。你需要根据此规范来搭建或配置你的AI智能体服务。

## 1. 交互逻辑概述

系统采用 **Webhook回调** 模式：
1.  **用户输入**：用户在前端“AI排课助手”输入自然语言提示词（例如：“3月25日有医疗峰会，王芳老师请假”）。
2.  **后端转发**：后端 `POST /api/ai/extract-constraints` 接收到请求后，将**提示词**和**当前排课上下文**（月份、班级列表、讲师列表）封装，转发给你配置的 `AI_AGENT_URL`。
3.  **智能体处理**：你的AI智能体接收请求，利用LLM（如GPT-4, Gemini等）理解语义，提取出结构化的**排课约束**。
4.  **返回结果**：智能体将提取结果按指定JSON格式返回给后端。
5.  **系统应用**：后端解析返回的JSON，将约束条件（如避开日期、讲师请假）应用到排课逻辑中。

---

## 2. 接口定义

你的外部智能体需要提供一个HTTP POST接口。

- **URL**: 自定义（例如 `https://your-agent.com/api/extract`），在系统前端或通过API配置。
- **Method**: `POST`
- **Content-Type**: `application/json`

### 请求结构 (Backend -> AI Agent)

系统发送给智能体的JSON数据包：

```json
{
  "prompt": "用户输入的自然语言提示词",
  "context": {
    "current_month": "YYYY-MM",        // 当前正在排课的月份
    "classes": ["班级A", "班级B"],      // 当前系统中的活跃班级名称
    "teachers": ["教师张", "教师李"],   // 系统中所有的讲师姓名
    "homeroom_teachers": ["班主任王"]   // 系统中所有的班主任姓名
  }
}
```

### 响应结构 (AI Agent -> Backend)

智能体必须返回的JSON数据包：

```json
{
  "success": true,               // 处理是否成功
  "constraints": {               // 提取出的约束对象
    "blocked_dates": [           // 1.需全校避开的日期
      {
        "date": "YYYY-MM-DD",    // 日期 (必填)
        "reason": "原因描述",     // 原因 (必填)
        "affected_classes": []   // 受影响班级，空数组表示所有班级 (可选)
      }
    ],
    "teacher_unavailable": [     // 2.讲师不可用/请假
      {
        "teacher_name": "王芳",  // 讲师姓名 (需匹配context.teachers)
        "dates": ["YYYY-MM-DD"], // 不可用日期数组
        "reason": "事由"         // (可选)
      }
    ],
    "homeroom_unavailable": [    // 3.班主任不可用/请假
      {
        "homeroom_name": "李婷婷", // 班主任姓名 (需匹配context.homeroom_teachers)
        "dates": ["YYYY-MM-DD"],   // 不可用日期数组
        "reason": "事由"
      }
    ],
    "preferred_dates": [],       // 4. (预留) 优先排课建议
    "merge_suggestions": [],     // 5. (预留) 合班建议
    "special_notes": "备注信息"   // 6. 其他备注
  }
}
```

## 3. 示例

### 场景演示

**用户输入**：
> "3月25日北京有大型医疗峰会，医疗班那天不能上课。另外王芳老师3月21号和22号要去上海出差，没法上课。"

**系统发送请求**：
```json
{
  "prompt": "3月25日北京有大型医疗峰会，医疗班那天不能上课。另外王芳老师3月21号和22号要去上海出差，没法上课。",
```

```json
{
  "success": true,

    "blocked_dates": [
      {
        "date": "2026-03-25",
        "reason": "北京大型医疗峰会",

    "teacher_unavailable": [
      {
        "teacher_name": "王芳",

      }
    ],
    "preferred_dates": [],
    "merge_suggestions": [],
    "special_notes": ""

}
```

## 4. 搭建建议

如果你使用 Dify、Coze 或自定义 Python 服务搭建智能体，核心逻辑如下：

1.  **Prompt设计**：
    > 你是排课辅助助手。请分析用户的自然语言指令，结合提供的上下文（班级列表、讲师列表），提取排课约束条件。
    > 请严格按照以下JSON格式输出结果，不要包含Markdown代码块标记。
    > [此处插入响应结构的JSON Schema]

2.  **实体映射与同名消歧**：
    > **姓名识别优先级**：
    > -   当用户输入姓名（如“王芳”）未明确指明身份时，请优先在 `context.teachers` 和 `context.homeroom_teachers` 中查找匹配项。
    > -   如果名字仅出现在 `teachers` 中，则归为 `teacher_unavailable`。
    > -   如果名字仅出现在 `homeroom_teachers` 中，则归为 `homeroom_unavailable`。
    > -   **重名处理**：如果名字同时存在于两者中，且用户未指定（如“王芳老师” vs “班主任王芳”），请根据上下文推断，或者(如果无法推断)两个列表都尝试匹配，或优先视为讲师（默认策略）。
    > -   提取班级名称时，尽量匹配 `context.classes` 中的标准名称。
    >
    > 日期必须转换为 `YYYY-MM-DD` 格式。

3.  **接口封装**：
    > 确保你的服务对外暴露 HTTP POST 接口，并能接收JSON body。

## 5. 如何配置

启动系统后端后，调用配置接口设置你的智能体URL：

```bash
curl -X POST http://localhost:5000/api/ai/config \
  -H "Content-Type: application/json" \
  -d '{"url": "http://your-ai-agent-ip:port/extract"}'
```

或者在前端界面（如果有开发配置页）进行设置。
